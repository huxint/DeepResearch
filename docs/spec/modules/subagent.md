# subagent 规约 —— 子 Agent 执行单元

## 1 职责　（强）
把"一次 LLM 调用"做成可靠执行单元：组装 prompt → 经 [[provider-pool]] 调用 → pydantic 校验输出 → 校验失败带错误上下文重试 → token 预算扣减。这是 [[kernel]] 的 `agent` 原语的执行体。

## 2 边界与非目标　（强）
- 不做编排 —— 交给 [[kernel]]。
- 不做缓存 —— 交给 [[journal]]（由 [[kernel]] 在调用本模块之前接入）。
- 不直接管 API Key/限流/熔断 —— 交给 [[provider-pool]]。

## 3 关键接口/契约　（中）
- `run(prompt, *, schema?, model?) -> 校验对象 | 文本`：执行一次子 Agent。给定 `schema` 时输出必须通过 pydantic 校验才返回校验对象；校验失败时**带上一次的错误上下文重试**，直至上限。

## 4 数据模型　（中）
- `SubagentSpec`：一次执行的输入规格（prompt、可选 schema、model、重试上限）。
- 校验错误上下文：上一次失败的校验错误信息，用于注入下一次重试 prompt。
- `BudgetTicket`：本次调用的预算扣减凭据（关联共享 token 预算池）。

## 5 不变式　（强）
1. 给定 schema 时，返回值必定是通过 pydantic 校验的对象。
2. 预算池超限 → 熔断，不再发起新调用。
3. 重试次数有上限，不会无限重试。

## 6 依赖　（强）
- 模块：[[provider-pool]]、cross-cutting（token 预算池）。

## 7 失败语义（弹性接缝）　（中）
- 输出 schema 校验失败 → 带错误上下文重试至上限 → 仍失败则上抛。
- 预算超限 → 熔断并上抛致命错误。
- 这是允许做错误处理的接缝之一。

## 8 验收标准　（强）
- **given** 一个会先产出非法、再产出合法输出的模型，**when** 带 schema 执行，**then** 触发重试并最终返回合法校验对象。
- **given** 始终非法的输出，**when** 执行，**then** 重试到上限后上抛。
- **given** 预算池已耗尽，**when** 执行，**then** 熔断、不发起调用并上抛。
- **given** 合法输出，**when** 带 schema 执行，**then** 返回对应的校验对象。

## 9 开放决策　（——）
- 重试上限的具体值。
- 错误上下文注入下一次 prompt 的格式与措辞。
- 预算扣减时机（调用前预扣 vs 调用后按实际 token 结算）。
