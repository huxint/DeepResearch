# Journal Implementation Plan

> Superpowers skills were not available in this Codex session, so this records the same writing-plans/TDD/verification loop for the `journal` module.

**Goal:** Build the `journal` module so agent calls can be cached by a stable `(prompt, params)` hash in an append-only SQLite store, enabling resume to hit unchanged prefixes and rerun from the first changed step.

**Architecture:** Create a focused `deep_research.journal` package. `models.py` owns pydantic boundary models and JSON payload types, `hashing.py` owns canonical JSON plus sha256 helpers, and `store.py` owns the SQLite Journal IO seam. Tests simulate kernel-style resume behavior without adding orchestration to the journal module.

**Tech Stack:** Python 3.14, sqlite3, pydantic, pytest, pyright strict, ruff.

## File Structure

- Create `src/deep_research/journal/models.py`: `JsonValue` and `JournalEntry`.
- Create `src/deep_research/journal/hashing.py`: stable canonical JSON, prompt fingerprinting, and journal hash helpers.
- Create `src/deep_research/journal/store.py`: SQLite-backed append-only `Journal`.
- Create `src/deep_research/journal/__init__.py`: public exports.
- Create `tests/journal/test_journal.py`: acceptance tests for §5 invariants and §8 behavior.
- Modify `docs/guide/01-build-order.md`: mark `journal` cells complete after T1 gate passes.

## Acceptance Mapping

- Stable `(prompt, params)` hash: `tests/journal/test_journal.py::test_hash_is_stable_for_canonical_prompt_and_params`.
- Unchanged prefix hits after restart: `test_resume_hits_unchanged_prefix_after_process_restart`.
- Changed middle step reruns from first miss onward: `test_resume_reruns_from_first_changed_step`.
- Duplicate append keeps lookup correct: `test_duplicate_append_keeps_lookup_correct`.
- Stored metadata remains queryable: `test_lookup_entry_returns_stored_metadata`.
