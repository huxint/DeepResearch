# Subagent Implementation Plan

> Superpowers skills were not available in this Codex session, so this records the same writing-plans/TDD/verification loop for the `subagent` module.

**Goal:** Build the `subagent` execution unit so one prompt can be sent through the provider pool, charged against the shared budget, optionally validated by a pydantic schema, and retried with validation context up to a fixed limit.

**Architecture:** Create a `deep_research.subagent` package. `models.py` owns pydantic boundary models for execution specs, validation retry context, budget tickets, and chat prompts. `runner.py` owns `Subagent`, depending only on a narrow `LLMProvider` protocol implemented by `ProviderPool`. Tests use a fake provider to drive deterministic invalid/valid outputs and budget behavior.

**Tech Stack:** Python 3.14, pydantic, provider-pool models, shared `BudgetPool`, pytest, pyright strict, ruff.

## File Structure

- Create `src/deep_research/subagent/models.py`: subagent specs, retry context, budget ticket, and chat prompt models.
- Create `src/deep_research/subagent/runner.py`: `Subagent` and `LLMProvider` protocol.
- Create `src/deep_research/subagent/__init__.py`: public exports.
- Create `tests/subagent/test_subagent.py`: acceptance tests for §5 invariants and §8 behavior.
- Modify `docs/guide/01-build-order.md`: mark `subagent` cells complete after T1 gate passes.

## Acceptance Mapping

- Invalid then valid schema output retries with context: `tests/subagent/test_subagent.py::test_schema_validation_failure_retries_with_error_context`.
- Always-invalid output raises after retry limit: `test_always_invalid_output_raises_after_retry_limit`.
- Exhausted budget aborts before provider call: `test_exhausted_budget_aborts_before_provider_call`.
- Valid schema output returns validated object: `test_valid_schema_output_returns_validated_object`.
- No-schema execution returns raw text: `test_without_schema_returns_text`.
