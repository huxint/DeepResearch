# Kernel Implementation Plan

> Superpowers skills were not available in this Codex session, so this records the same writing-plans/TDD/verification loop for the `kernel` module.

**Goal:** Build the workflow kernel with deterministic `agent`, `pipeline`, and `parallel` primitives. `agent` must consult Journal before subagent execution and enforce a concurrency cap. `pipeline` must be no-barrier per item. `parallel` must aggregate results while degrading failed items to `None`.

**Architecture:** Create a `deep_research.kernel` package. `models.py` owns public orchestration models and type aliases. `orchestrator.py` owns `Kernel`, narrow protocols for subagent and Journal dependencies, and the default concurrency limit. Tests use fake subagents plus a real Journal to prove cache behavior and scheduling semantics.

**Tech Stack:** Python 3.14, asyncio TaskGroup, pydantic, journal, subagent protocol, pytest, pyright strict, ruff.

## File Structure

- Create `src/deep_research/kernel/models.py`: `AgentResult`, `OrchestrationContext`, `Stage`, and `Thunk`.
- Create `src/deep_research/kernel/orchestrator.py`: `Kernel`, dependency protocols, concurrency limit helper.
- Create `src/deep_research/kernel/__init__.py`: public exports.
- Create `tests/kernel/test_kernel.py`: acceptance tests for §5 invariants and §8 behavior.
- Modify `docs/guide/01-build-order.md`: mark `kernel` cells complete after T1 gate passes.
