# Verifier Implementation Plan

> Superpowers skills were not available in this Codex session, so this records the same writing-plans/TDD/verification loop for the `verifier` module.

**Goal:** Build citation verification so each citation is fetched through `mcp-tools`, checked from multiple independent perspectives via `kernel.parallel`, and aggregated into a verdict that marks unsupported citations as citation errors.

**Architecture:** Create a `deep_research.verifier` package. `models.py` owns citation, per-perspective verdict, aggregate verdict, and summary models. `checker.py` owns `Verifier`, dependency protocols for kernel/tools, default perspectives, and majority aggregation. Tests use real `Kernel.parallel` plus fake fetch tools to verify fake citations, supported citations, batch statistics, and single-perspective failure degradation.

**Tech Stack:** Python 3.14, kernel parallel, mcp-tools fetched docs, pydantic, pytest, pyright strict, ruff.

## File Structure

- Create `src/deep_research/verifier/models.py`: `Citation`, `PerspectiveVerdict`, `Verdict`, and `VerificationSummary`.
- Create `src/deep_research/verifier/checker.py`: `Verifier`, default perspectives, and aggregation.
- Create `src/deep_research/verifier/__init__.py`: public exports.
- Create `tests/verifier/test_verifier.py`: acceptance tests for §5 invariants and §8 behavior.
- Modify `docs/guide/01-build-order.md`: mark `verifier` cells complete after T1 gate passes.

## Acceptance Mapping

- Unsupported citation is marked as citation error: `tests/verifier/test_verifier.py::test_fake_unsupported_citation_is_marked_as_citation_error`.
- Supported citation passes: `test_supported_citation_passes`.
- Batch verification reports unsupported count: `test_verify_all_reports_unsupported_citation_count`.
- Single perspective failure does not block aggregation: `test_single_perspective_failure_does_not_block_aggregation`.
- Public verdict model exists: `test_verdict_model_is_public`.
