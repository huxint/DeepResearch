# Eval Implementation Plan

> Superpowers skills were not available in this Codex session, so this records the same writing-plans/TDD/verification loop for the `eval` module.

**Goal:** Build dataset evaluation so a research pipeline can be run over text cases, judged per case, summarized with accuracy and cost, audited against human labels, and continue after single-case failures.

**Architecture:** Create a `deep_research.eval` package. `models.py` owns evaluation cases, per-case results, summaries, judge decisions, manual judgements, and audit models. `runner.py` owns `EvalRunner`, protocols for pipeline/judge/cost tracker, a deterministic judge implementation, and judge consistency audit. Tests use a fake pipeline and cost tracker to verify per-case results, summary aggregation, reproducibility, failure continuation, and per-case cost deltas.

**Tech Stack:** Python 3.14, pipeline report model, provider-pool-compatible cost tracking, pydantic, pytest, pyright strict, ruff.

## File Structure

- Create `src/deep_research/eval/models.py`: evaluation, result, and audit pydantic models.
- Create `src/deep_research/eval/runner.py`: `EvalRunner`, judge/cost protocols, deterministic judge, and audit helper.
- Create `src/deep_research/eval/__init__.py`: public exports.
- Create `tests/eval/test_eval.py`: acceptance tests for §5 invariants and §8 behavior.
- Modify `docs/guide/01-build-order.md`: mark `eval` cells complete after T1 gate passes.

## Acceptance Mapping

- Eval outputs per-case and summary: `tests/eval/test_eval.py::test_run_eval_outputs_per_case_and_summary`.
- Judge audit reports human agreement: `test_judge_audit_reports_human_agreement`.
- Single-case failure is marked wrong and batch continues: `test_single_case_failure_is_marked_wrong_and_batch_continues`.
- Deterministic judge is reproducible: `test_deterministic_judge_is_reproducible`.
