# 质量门（Quality Gates）+ 完成定义

三层门由窄到宽。前一层不过，不进下一层。门检查的是**功能是否成立**，不是有没有达到某个数字——本项目不为任何指标数字开发。

## T1 模块门（每个模块收尾时）

- `pyright --strict` 零报错。
- 该模块验收测试（规约 §8）全绿。
- 不变式（规约 §5）有测试覆盖。
- `ruff` lint + format clean。
- **防御性 try 审查**：每处 try/except 必须落在 `02-coding-conventions.md` 列的四个弹性接缝内，否则删除或改为上抛。

## T2 集成门（波次边界）

- **tracer-bullet 端到端**：对一道题端到端产出带引用报告。
- **Journal 续跑**：跑到一半杀进程，resume 命中未变更前缀缓存、不重跑已完成步骤。
- **号池韧性**：注入 429/超时，验证熔断换道 + 指数退避 + 多路扇出不中断。

## T3 系统门（系统成型后，功能性检查）

| 门 | 通过条件 |
|---|---|
| 引用门 | 报告里每条引用都经 Verify 核验，不被来源支持的引用被过滤 |
| 评测门 | 能在 GAIA 文本子集等数据集上端到端跑完并产出结果 |
| 成本门 | 共享 token 预算池在超限时能正确熔断 |
| 时延门 | 多路并行检索明显快于串行 |
| 回归门 | 在 WebWalkerQA 等数据集上重复跑不崩、结果稳定 |

## 原则

门只验证**功能是否成立**：能不能端到端跑通、引用能不能溯源、能不能续跑、超限能不能熔断。不为任何指标数字开发，先把系统做到能跑、能用。

## 完成定义 (Definition of Done)

- `01-build-order.md` 构建状态表中，所有模块的五列全部勾选完成。
- T3 系统门各项功能性检查通过。
